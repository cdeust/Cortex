# Wiki Sync — Browsable Memory Projection

## Keywords
wiki, sync wiki, render wiki, browse memories, markdown export, memory documentation, inspectable memory, human-readable memory, mermaid charts, memory wiki

## Overview

Generate a **read-only Markdown projection** of Cortex memory state under `~/.claude/methodology/wiki/`. PostgreSQL remains the single source of truth — the wiki is a deterministic rendering for inspectability and adoption (browsable in any editor, renders Mermaid charts inline in Claude Code CLI).

**Use this skill when:** the user wants to browse what Cortex remembers without SQL, share memory state with non-developers, or inspect schemas/causal chains visually.

**Do NOT use for:** improving recall/reranking (this is a view, not a memory mechanism), or as a place to edit memories (edits will be overwritten on the next sync).

## Workflow

### Sync everything

```
cortex:wiki_sync({})
```

Writes:
- `~/.claude/methodology/wiki/INDEX.md` — global domain table
- `~/.claude/methodology/wiki/<domain>/INDEX.md` — per-domain page

Returns `{written, skipped, pruned, root, domains, pages}`.

### Sync a single domain

```
cortex:wiki_sync({
  "domain": "cortex"
})
```

### Dry run (plan without writing)

```
cortex:wiki_sync({
  "dry_run": true
})
```

## Output layout

```
~/.claude/methodology/wiki/
  INDEX.md                          # global: domains list
  <domain-id>/
    INDEX.md                        # domain summary, top categories
    schemas/<schema_id>.md          # (slice 2)
    chains/<chain_id>.md            # (slice 4)
```

Every file carries a banner: `Generated by Cortex wiki_sync — edits will be overwritten.` Anything under the wiki root not produced by the latest sync is pruned automatically.

## Tips

- **Idempotent**: re-running with no memory changes is a no-op (`written=0, skipped=N`).
- **Out of band**: never invoked from `consolidate` or the recall hot path. Safe to call any time.
- **Browse with anything**: Obsidian, Logseq, `mdbook`, plain `grep`, or `cat` in the terminal.
- **Mermaid renders inline** in Claude Code CLI — schema graphs and causal chains will appear as diagrams in later slices.
